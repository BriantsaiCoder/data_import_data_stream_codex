1 系統概觀與目標

一支長時間運行的背景服務，把半導體 DCT(Direct Circuit Test)測試結果檔從 FTP 或 LocalImportRoot 自動匯入 MySQL。啟動後進入無限迴圈，常駐三條工作執行緒，每約 7.2 分鐘輪詢一次待匯入資料（非一次性 ETL 工具）。

🎯 核心目的

FTP / Local 測試檔 → 解析 → 驗證 → 批次寫入 MySQL

🔁 運行模式

無限迴圈 + 三常駐執行緒(死亡自我復活)

🚩 完整性判定

4-bit check_status 旗標，逐子項加權比對

📧 失敗處置

回寫 mail=1 + 下輪重做 + 寄通知信

品質目標在架構中的落實
資料正確性check_status 位元完整性判定(⑩)、欄位 Compare* 驗證、統計值清洗(⑪)
長時穩定運行三執行緒自我復活監督(⑦)、寄信前 Ping、Mutex 保護日誌(⑭)
跨環境單一 binary靠執行機本機 IP 切 Dev/Prod，不需重編譯(⑧)
可安全試跑DryRun 影子模式:解析比對照常、所有對外寫入轉 no-op(⑧)
無終端使用者登入——這是背景批次服務。利害關係人:產線/測試工程師(資料消費者)、維運(部署監控執行機)、接手開發者(本頁主要讀者)。

7 執行期視圖：三執行緒模型

Main(Program.cs:29)無限迴圈常駐三條工作執行緒(Program.cs:97-99)。迴圈偵測死亡執行緒並透過 RestartWorker(Program.cs:198)以 Interrupt → new → Start 重啟;Thread.Abort() 在 net5+ 已停用，僅保留註解。

ImportTesterMode
Tester 模式

主要測試資料:RawData / Tester / FailPin / RecoveryRate / MultiSpec，並計算 check_status。

ImportUiStatusMode
UiStatus 模式

獨立處理 UI 狀態，寫 ui_status 表。不在 check_status 旗標內。

ImportTsmcMode
TSMC 模式

TSMC IEDA 對接，固定寬度解析。不讀 DB 的 DB_Key 清單，直接掃 FTP 目錄。

🐞 單位陷阱:Thread.Sleep(432000)(Program.cs:145)註解寫「2H」，但 432000 毫秒 = 432 秒 = 7.2 分鐘。實際輪詢週期是 7.2 分鐘，不是 2 小時;錯誤時 Thread.Sleep(30000)(:155)退避 30 秒再重試。
Main 啟動只輸出 HOST/USER 與 PASSWORD set/unset，不印 PASSWORD 明文;新增啟動輸出要維持遮罩(⑱ 風險 1/2)。

8 環境與資料來源切換

靠執行機本機 IPv4 硬編碼判定，不是 build configuration(GetEnvironment Program.cs:224 / GetLocalIPAddress :243)。

環境路由

productionIps = {"10.16.92.67","10.16.92.68"}(Program.cs:231)命中即 Prod，否則 Dev。連線鍵以字串串接 ${Environment}Host 對應 App.config。

FTP 根路徑分環境

Dev /DCT_Log/DCT_DB_DATA_Dev/;Prod /DCT_Log/DCT_DB_DATA/。

⚠️ Release build 不會自動指向 Prod——環境選擇是 runtime 邏輯，純看執行該程式的機器 IP(換機/換 IP 即失效，⑱ 風險 6)。

資料來源(FTP / Local)

ImportFileSourceFactory.Create() 依 ImportSource 建立來源——預設 Ftp;Local 時從 LocalImportRoot 讀同一套相對路徑，成功後依 LocalSuccessAction 決定 Delete(預設)或移到 Imported/。

影子驗證模式(DryRun):App.config 設 DryRun=true(預設 false、缺 key 亦 fail-safe OFF;RuntimeMode.cs:24)時整條 ETL 變唯讀 —— DB 寫入(含 ⑬ cascade delete，DBmysql.cs:98)、FTP 刪/改名、Local 搬檔(ImportFileSource.cs:221,237,316,336)、寄信(EmailModels.cs:25)全部轉 no-op，只解析/比對並保留 mail_temp.txt 供比對;SELECT 仍照常。check_status / import_status 照常計算，只是結果不落地。用途:cutover 影子試跑與資料污染回滾防護。

11 資料庫存取層 + 匯入引擎

DbAccess.cs 控制表

控制表 db_key / db_key_ui_status;計算 check_status 公式;失敗通知清單由 mail_temp.txt 經 SelectFailDbKeyFromFile() 讀回。

DatabaseService.cs typed-only

ExecuteQuery 回 DbQueryResult;ExecuteCommand 回 DbCommandResult。CheckDatabaseAndTableExists 只檢查存在性、不自動建表。

DBmysql.cs typed-only

每次 new MySqlConnection;MySqlConnectionManager 單例連線字串(池 Min5/Max100)。FilterSqlCommand 為 S2 前殘留防線，非主要安全控制。

DbObject.cs 回應契約

DbSqlRequest = Query + Parameters;DbQueryResult = JArray Data + Error;DbCommandResult = AffectedRows + InsertId + Error。

⚠️ 錯誤以 Error 字串傳遞，不拋例外:呼叫端必須主動 string.IsNullOrEmpty(response.Error) 檢查，否則錯誤被靜默吞掉。現況是 Dapper 直連 MySQL，沒有外部 HTTP API Service、沒有自動建表、沒有 ORM 真實事務——DB result surface 為 typed-only。

匯入引擎(FileProcess.cs)

全部 Import* 集中於此，是最大的單檔、扇入極高(動它前先列依賴方)。

參數化 chokepoint

INSERT/DELETE 值走 Dapper parameters;ExecuteInsert(FileProcess.cs:1414)統一接 DynamicParameters，以 identifier guard 擋非法 table/column name。新增寫入路徑不得繞過 ExecuteInsert。

批次切割(寫死在程式)

lots_statistic:cut=10000/test_count;recovery_rate/lots_result:5000/批;fail_pin:50/批。大型批次已改 StringBuilder，小表單筆路徑仍 values += ...。

統計值清洗

ValidateAndConvertStatisticValue(FileProcess.cs:1697)把 -1.#IND、1.#QNAN 轉為 0。

其他

tester_status 只取第一列;「事務回滾」實為 best-effort cascade DELETE(見 ⑬)。

13 錯誤回滾的真相 best-effort

沒有真實 DB transaction 保護整個匯入單元。失敗時以串接 DELETE 嘗試清掉已寫入資料，有兩個已知缺口。

① lots_result 失敗 → 殘留 orphan

lots_info 與 lots_statistic 已先寫入，若 lots_result 失敗，DELETE 不一定清得乾淨。

② MultiSpec 多 site 失敗 → 部分匯入

後段 site 失敗，前段已成功的 site 不回滾。

此外，無 FK cascade 的子表(tester_sw_version / ieda_content / ui_status，見 ⑫)即使父表被刪也不會自動連帶，更依賴 app 端 DELETE 順序正確。也沒有通用的重試/狀態復原機制:唯一重試是 TsmcIeda.GetNetNameList 的一次性遞迴。狀態「復原」靠 import_status 標記 + 失敗回寫 mail=1 + 7.2 分鐘迴圈下輪重撈重做。

14 橫切關注點

同步模型:明確同步阻塞。importer 回 ImportResult 由 worker thread 直接呼叫;DB API 同步 ExecuteQuery/ExecuteCommand;WriteToLog 刻意用同步 StreamWriter.WriteLine(避免 fire-and-forget 丟日誌)。非「真 async」;要提升 I/O 併發需另設計 async DB/FTP path。

NotificationService

收件人從 dct_import_mail_list.ini 的 [mail_list] 讀;發信時窗硬編碼——只在週一 08:00–08:09 回報平安。⚠️ ui_status 失敗通知被註解停用。

EmailModels

純 SMTP 發送器，server/from 由 App.config 指定;寄信前先 Ping，失敗即放棄(避免卡 timeout)。內網匿名 relay、無 SMTP auth/TLS;ToList null guard 不 NRE。

WriteToLog 最高扇入

根目錄 DataImportLogRoot(預設 C:\temp);DataImportLogRetentionDays 預設 90 天(retention，非 size rotation);跨程序 Mutex(逾時 30 秒吞例外、日誌可能靜默遺失)。⚠️ UTF-8 BOM 不一致:data/check log 用 BOM，WriteToMailTemp 無 BOM。

CalculateSPC

⚠️ avg2 存的是平方平均 E[x²]，非變異數;呼叫端須 avg2 - avg² 再開根號(母體標準差，除 N)。根號內為負時 Sqrt(負)=NaN、ToDecimal(NaN) 拋例外被外層 catch 吞掉。
